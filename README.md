# Сравнительный анализ моделей детекции (COCO 2017)

Автоматизированный пайплайн для проведения инференса и оценки метрик (mAP, AR, Speed) моделей детекции объектов двух семейств (LibreYOLO9/YOLOX/DEIM и LibreDFINE/RFDETR/RTDETR) в формате ONNX.

## Быстрый старт

Выполните следующие шаги для развертывания окружения и запуска бенчмарка:

### 1. Клонирование репозитория
```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd <ИМЯ_ПАПКИ_РЕПОЗИТОРИЯ>
```

### 2. Создание и активация виртуального окружения
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 4. Подготовка датасета COCO
Поместите валидационную выборку COCO2017 в корень проекта так, чтобы структура директорий выглядела следующим образом (пути можно изменить в конфигурациях):
```text
datasets/
└── coco/
    ├── annotations/
    │   └── instances_val2017.json
    └── val2017/
        ├── 000000000139.jpg
        └── ...
```

### 5. Запуск пайплайна YOLO / DEIM
Автоматический скрипт скачает все необходимые модели с Hugging Face и разложит их по папкам, а затем проведет инференс:
```bash
cd yolo_deim
python3 download_weights.py
python3 run_pipeline.py
cd ..
```

### 6. Запуск пайплайна DETR / DFINE
Запустите главный скрипт ветки DETR. Он автоматически скачает модели, проведет инференс, посчитает метрики и замерит скорость:
```bash
python3 detr_dfine/main.py
```

### 7. Объединение результатов
Сведите результаты обеих веток в единый бенчмарк:
```bash
python3 results/merge_reports.py
```

---

## Результаты работы

После завершения всех этапов в директории `results/` автоматически появятся:
* `benchmark_all_models.csv` — итоговая сводная таблица метрик (mAP50-95, mAP50, AR), параметров и времени кадра (ms) для всех моделей.
* Папка `pr_curves/` — сгенерированные графики **Precision-Recall Curve** для каждой модели.
* Промежуточные CSV-файлы с частичными метриками (`detr_dfine_metrics.csv` и т.д.).

Сырые предсказания моделей сохраняются в формате `.json` в локальных папках `predictions/` внутри каждой ветки.
